⚙️ Operación y Despliegue

Esta sección describe cómo se opera, mantiene y despliega este sitio web utilizando servicios administrados de AWS.

Modelo operativo

El sitio funciona bajo un modelo serverless, sin servidores que administrar ni sistemas operativos que mantener. La operación diaria se centra en el contenido y la configuración de la CDN.

  • No existen instancias EC2 ni procesos en ejecución.
  • No se requieren parches de sistema operativo.
  • La disponibilidad es gestionada por AWS.

Despliegue

El proceso de despliegue se inicia al ejecutar un git push sobre la rama principal del repositorio. GitHub Actions detecta el cambio y ejecuta un workflow automatizado. Durante la ejecución, GitHub Actions se autentica en AWS mediante OIDC, asumiendo un IAM Role previamente configurado con una relación de confianza restringida al repositorio y rama específicos. El pipeline sincroniza los archivos del sitio hacia Amazon S3, que actúa como origen privado del contenido. Finalmente, Amazon CloudFront distribuye el contenido actualizado desde el edge, asegurando entrega segura y eficiente a los usuarios finales.

Flujo de despliegue

El despliegue del sitio es simple, controlado y reproducible. Cualquier cambio sigue el siguiente flujo:

  1. Edición de archivos HTML/SVG en entorno local.
  2. Subida de archivos al bucket S3 privado.
  3. Invalidación selectiva de caché en CloudFront.
  4. Propagación global automática del contenido.

Control de versiones

El contenido del sitio se gestiona como archivos estáticos. Los cambios son pequeños, auditables y fáciles de revertir.

Gestión de caché

CloudFront utiliza caché para mejorar rendimiento y reducir accesos al origen. Cuando se realizan cambios, se ejecuta una invalidación manual para asegurar que los usuarios reciban la versión actualizada.

  • Invalidación puntual por archivo.
  • No se requiere reinicio de servicios.
  • Propagación global en minutos.

Monitoreo y observabilidad

Al utilizar servicios administrados, la plataforma se beneficia de métricas nativas de AWS.

  • Métricas de tráfico y errores en CloudFront.
  • Logs de acceso disponibles para auditoría.
  • Alta disponibilidad gestionada por AWS.

FinOps y Control de Costos

Como parte de las mejores prácticas de operación en la nube, se ha implementado una estrategia preventiva para la gestión financiera utilizando AWS Budgets.

  • Zero-Spend Budget: Alertas configuradas para notificar inmediatamente si el gasto proyectado supera la capa gratuita mensual de AWS ($0.01 USD).
  • Monitoreo Proactivo: Permite detectar picos inusuales de tráfico o errores de configuración antes de que generen un impacto financiero.
  • Visibilidad Continua: Integración de las alertas de facturación y notificaciones de CloudWatch para un control granular de los costos.

Futuras mejoras

  • Automatización del despliegue mediante CI/CD.
  • Integración con Git para control de cambios.
  • Implementación de AWS WAF para mayor protección.
  • Dominio personalizado con AWS Certificate Manager.

Operar menos infraestructura permite enfocarse en entregar valor.
